# Invest Share Monitor

一个面向小型合伙投资场景的前端账本系统：将股票账户按“基金份额”方式管理，自动完成成员收益分摊、当日收益计算与持仓看板展示。

## 功能概览

- 双页面结构：
  - `index.html`：展示页（给成员/客户看）
  - `operation.html`：操作页（管理员维护）
- 成员管理：新增/删除成员，按份额参与收益分配
- 资金流管理：入金/出金自动按当时净值折算份额
- 股票交易记录：买入/卖出驱动持仓、成本、现金联动计算
- 持仓行情更新：支持按股票代码自动刷新现价（失败可手动录入）
- 收益体系：
  - 累计收益（总收益、成员收益）
  - 当日收益（基于日内基线，排除资金流影响）
- 操作日志与备注：保留管理动作痕迹，便于复盘
- 本地优先存储：
  - 默认 `localStorage`
  - 支持连接本地 JSON 文件并持久化读写
  - 支持导入/导出 JSON 备份

## 核心计算规则

### 1) 份额机制

- 入金份额：`入金金额 / 入金时净值`
- 出金份额：`出金金额 / 出金时净值`

成员收益按份额而不是按人头分配，避免“新入金共享历史收益”问题。

### 2) 总收益

- 成员净入金：`累计入金 - 累计出金`
- 成员资产：`成员份额 × 当前净值`
- 成员总收益：`成员资产 - 成员净入金`
- 基金总收益：`所有成员总收益之和`

### 3) 当日收益基线

当日收益基于“昨日总资产 + 今日净入金”作为基线，避免把入金误算为盈利：

- 今日净入金：`今日入金 - 今日出金`
- 当日收益：`当前总资产 - (昨日总资产 + 今日净入金)`

## 技术栈

- 纯前端：HTML + CSS + Vanilla JavaScript
- 无后端依赖、无打包步骤
- 数据核心模块：
  - `storage.js`：状态结构与本地持久化
  - `ledger-core.js`：计算引擎（份额、收益、持仓）
  - `app.js`：页面渲染、交互事件、文件读写
  - `trade-optimizer.js`：交易录入与股票搜索补全

## 快速开始

### 方式一：直接打开

双击打开 `index.html` / `operation.html` 即可使用。

### 方式二：本地静态服务（推荐）

```bash
cd /path/to/invest_monitor
python3 -m http.server 8080
```

浏览器访问：

- `http://localhost:8080/index.html`
- `http://localhost:8080/operation.html`

## 部署（GitHub Pages）

1. 推送代码到 GitHub 仓库
2. 仓库 `Settings` -> `Pages`
3. `Build and deployment` 选择 `Deploy from a branch`
4. 选择分支（如 `main`）和目录（通常 `/root`）
5. 保存后等待部署完成
6. 访问 `https://<your-username>.github.io/<repo-name>/index.html`

## 数据持久化与备份

在操作页「数据管理」中可：

- 连接本地文件（后续自动记住并优先读写该文件）
- 刷新/保存/另存
- 导入 JSON / 导出 JSON
- 清空全部数据（带确认口令）

建议为真实账本单独准备一个不入库的 JSON 文件，例如 `fund-ledger-YYYYMMDD.json`。
